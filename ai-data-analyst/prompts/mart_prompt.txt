## Rules
1. Use CREATE OR REPLACE VIEW {database}.{view_name} AS ...
2. View name format: mart_<descriptive_name> (e.g., mart_age_product_monthly). MUST be ASCII-only, no Korean characters.
3. Include useful derived columns: age_group, month, quarter, etc.
4. For age groups: CASE WHEN CAST(age AS INT) < 20 THEN '10대' WHEN < 30 THEN '20대' ... ELSE '60대+' END
5. For time: DATE_TRUNC('month', DATE_PARSE(purchase_date, '%Y-%m-%d'))
6. Only include completed orders unless explicitly asked otherwise.
7. Pre-aggregate where appropriate for the mart's purpose.
8. Use Athena-compatible SQL (Presto/Trino syntax).
9. Return ONLY valid JSON, no markdown fences.
